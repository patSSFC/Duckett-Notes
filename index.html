Writing Links

You Always Create links using the <a> tag and the href attribute. Anything inside the tags, the user can click on. Have a question? Google it.

Text between the opening of a tags is known as link text. Link text should describe the location where the user will be taken as opposed to "click here"

Linking to other sites

When you link to a different website, the URL in the href attribute will be the full web address for the site AKA the absolute URL.

Linking to other pages on the same site/dir

When you link to pages within the same site, you do not need to specify the domain name in the URL. Can use shorthand known as a relative URL. If all the files of a web page are within the same folder/dir then the href attrb is simply the name of the file.

Opening Links in a New Window

To open a link in a new window, you must use a new attrib called 'target'. To open in a new windows the value must be set to '_blank'. Most common reason to open a link in a new window is if the link is referring to an external web page. The World Leader!

Email Links

To create a link that starts up the users email program you use the <a> tag using the 'href' attribute. However the href attrib would begin with 'mailto:'

Linking to a Specific Part of the Same Page

At the top of a loong page, you may want to add of list of contents that links to the corresponding section lower down on the page. Or vice versa, going from somewhere down below back to the top. Before you can link to a specific part of a page, you must identify the sections that the link will lead to. This is done using the id attribute.

The id attribute can be used for every HTML element. Value of id should either start with a letter or an underscore (not a number or some othe character). And no two id's should have the same value

To link to an element that uses the id attrb, you use the <a> tags with the value of the href attrib set to the value of the id attrib associated with the section you wish to link to. The value of the href must start with the # symbol and then the value of the id attrib.

Linking to a Specific Part of Another page

If you want to link to a specific part of a different web page, you would use a similar technique as linking to one within your site. As long as the page you are linking to uses the id attribute you can simply add the same syntax to the end of the link for that page. W3C Footer?

Chapter: Forms(!!!)

HTML borrows the concept of a form to refer to different elements that allow you to collect information from visitors to your site. They are used for a whole ton of things (which you've already come across...skip the filler!!)

How Forms Work

  1. A user fills in a form and then presses a button to submit the info to a server
  2. The name of each form control is sent to the server along with the value the user enters or selects
  3. The server processes the information using a programming language like PHP, C#, VB.net or Java. It may also store the info into a Database
  4. The server creates a new page to send back to the browser base on the information received

It is important to give each form a unique identifier so the server can differentiate between the different types of input that the user is submitting. These are called name/value pairs.

Form Structure

Forms live inside the <form> element. This element should always carry the 'action' attribute and will usually also include the 'method' and 'id' attribute as well.

Every form requires the action attribute. It's value is the URL for the page on the server that will receive the information when it is submitted.

Forms can be sent using one of two methods: 'get' or 'post'. The 'get' method add the values from the forms to the end of the URL specified in 'action'. 'get' is ideal for short forms (such as search boxes) or when you are just retrieving data from the web server (not sending data that should be added or deleted from a DB)

With the 'post' method, the values are sent in what are known as HTTP Headers. As a rule of thumb, you should use the 'post' method if your form: allows users to upload a file, is very long, contains sensitive data (passwords), adds info or deles info from a DB. If 'method' is not set then default is get.

'id' attribute is looked at in more detail on pg. 183 but the value is used to identify the form distinctly from other elements or forms on the page. And is used by scripts to validate fields in forms.

Text Input

The <input> element is used to create several different form types. The value of the 'type' attribute determines what kind of input it will create. <input> is an empty element.

When the 'type' attrib is set to 'text', it creates a single-line text input. Again, the server needs to know which form controls each piece of data it receives. Therefore, every form control requires a 'name' attribute. This name value is sent to the server along with the associated data.

Information About Your Pages

The <meta> lives within the <head> element and contains information about your page. The user doesn't see any of this info but it fulfills a number of purposes such as telling search engines about your page, who created it, and whether it is time sensitive (if it is, the page can be set to expire)

The <meta> element is an empty element so it does not have a closing tag. The information is carred by attributes. Most common attributes are 'name' and 'content' which tend to be used together. The value of the 'name' attribute is the name of the property which you want to address and the 'content' is the value that you want to give to this property.

description
This contains a description of the page. Used by search engines to figure out what your page is about. Maximum of 155 characters. Sometimes displayed in SE results.
keywords
This contains a list of comma-seperated words that a user might search on to find the page. This no longer has any noticeable effect on how search engines index your site.
robots
INdicates whether SEs should add this page to their search results or not. A value of 'noindex' means it would not be added. A value of 'nofollow' indicates that the page should be displayed in the search resutls but none of the pages it links to.

The <meta> element also use the 'http-equiv' and 'content' attributes in pairs. You can use this name/value pair to set the author of the page, whether it should be cached, and the date/time that the cached page should expire.

How Flash Works

You can create your own Flash movies using the Flash authoring platform ($$$). When a movie is created using the Flash authoring envior it is saved in a .fla file extension. To use this file on your web page it must be converted into a different format, known as SWF (file extension: .swf).

In previous version of HTML, when you exported a .fla file to the .swf format, Flash would create HTML code that you could embed into your webpage. Traditionally, it would use the <object> and <embed> elements. However, now it is more common to use JavaScript to implement Flash movies into your page.

Adding a Flash Movie to Your Web Page

The most popular way of adding Flash into your page is using JavaScipt. There are several scripts that allow you to do this without understanding JS very well (which you will in time!!). The script we are going to look at is called the SWFObject. You can obtain a copy of it for free from Google. One advantage to using this technique is that it allows browsers to show altrenative content for users who are not capable of showing Flash.

The technique we are focusing on uses a <div> element to create a space where the Flash movie should sit. It will use an 'id' attribute whose value is used by the SWFObject script. Inside the <div> element you can place an alternative description for users who cannot play Flash, like a description of the movie or even a still from it.

The SWFObject script will check to see if the user's browser can play the Flash movie, if it can, the script will replace the content of the <div> element with the .swf file. The <script> elements are placed in the 'head' section of the page.
The 'type' attribute is used in the <script> element to indicate that the script inside is written in JavaScipt. The 'src' attribute tells the browser where to find the script (in this case, hosted on a Google AJAX server).
The second <script> element is used to tell the browser about the Flash movie, as well as which element it should replace. This element is actually telling the SWFObject script five pieces of information, which are in the brackets:

  1. The location of the .swf file
  2. The element thta the Flash movie should replace, specified by the 'value' of the 'id' attribute on the <div> elelent
  3. The width of the Flash movie
  4. The height of the Flash movie
  5. The minimum version of the Flash player needed to view the movie

CSS uses rules to associate with elements on an HTML page. These rules govern how the specified elements should be displayed. A CSS rule contains two parts: a selector and a declaration.

Controlling the Position of Elements

CSS has the following positioning schemesthat allow you to control the layout of a page: normal flow, relative positioning, and absolute positioning. You specify the positioning scheme using the 'position' property in CSS. You can also folat elements using the 'float' property.

Normal Flow

Every block-levelelement appears on a new line, causing each item to appear lower down the page than the pervious one. Even if you specify the width of the boxes and there is space for two elements to sit side-by-side, they will not appear next to each other. This is the default behavior (unless you tell the browsers to do something else).

Relative Positioning

This moves an element from the position it would be in normal flow, shifting it to the top, right, bottom, or left of where it would have been place. The second paragraph has been pushed down and right from where it would otherwise have been in normal flow.This does not affect the position of surrounding elements; they stay in the position they would be in normal flow.

Absolute Positioning

This position the element in relation to its containing elements. It is taken out of normal flow, meaning that it does not affect the position of any surrounding elements (as they simply ignore the space it would have taken up.) Absolutely positioned elements move as users scroll up and down the page.

Fixed Positioning

A form of absolute positioning, fixed positioning places the element in relation to the browser window, as opposed to the containing element. Elements with fixed positioning do not affect the position of surrounding elements and they do not move when the user scrolls up or down the page.

Floating Elements

Floating an elements allows you to take that element out of normal flow and psotion it to the far left or right of a containing box. THe floated element becomes a block-level element around which other content can flow.

Normal Flow

In normal flow, each block-level element sits on top of the next one. Since this is the default way in which browseres treat HTML elements, you do not need a CSS property to indicate that elements should appear in normal flow, but the syntax would be using a property of 'position' with a value of 'static'

Relative Positioning

Relative positioning moves an elements in relation to where it would have been in normal flow. For example, you can move it 10 pixels lower that it would have been in normal flow or 20% to the right.

You can indicate that an element shoud be relatively positioned using the 'position' property with a value of 'relative'. You then use the offset propertires ('top' or 'bottom' and 'left' or 'right') to dinciate how far to move the element from where it would have been in normal flow. Values of offset are usually given in pixels or ems.

Absolute Positioning

When the 'position' property is given a value of 'absolute', the box is takene out of normal flow and no longer affects the psotion of other elements on the page. (The other elements simply act like it is not present). The same offset properties can be used to specify where the element should appear in relation to its containing element.

By default most browsers add a margin to the top of the <h1> element. This is why there is a gap between the top of the browser and the box containing the <h1> element. If you want to remove the margin you must specify it in CSS.

Fixed Positioning

Fixed positioning is a type of absolute positioning that requires the 'position' property to have a value of 'fixed'. It postions the element in relation to the browser window. Therefore, it would stay in its place when the user scrolls up or down.

Overlapping Elements

When you use relative, fixed, or absolute positioning, boxes can overlap. If they do, the elements that appear later in the HTML code sit on top of those that are earlier in the page. If you want to control which element sits on top, you can use the 'z-index' property.

The value for 'z-index' is a number, and the higher the number the closer that element is to the front/top. It is sometimes reffered to as the stacking property.

Floating Elements

The 'float' property allows you to take an element in normal flow and place it as far to the left or right of the containing element as possible. Anything else that sits inside the containing element will flow around the element that is floated.

When you use the float property, it is also important to set the 'width' to indicate how wide the element should be. Otherwise, the box is likely to take the full width of the containing element like it would in normal flow.

Using Float to Place Elements Side-By-Side

A lot of layouts place boxes next to each other. The 'float' property is commonly used to achieve this. When elements are floated, the height of the boxes can affect where the following elements sit.

Setting the height of the paragraphs to be the same height as the tallest paragraph would solve this issue. But it is rarely suited to real word designs where the amount of text in a paragraph or column may vary. It is more common to use the clear property.

Clearing Floats

The 'clear' property allows you to say that no element (within the containing element) should touch the left or right-hand sides of a box. It can take the following values:

left
The left-hand side of the box should not touch any other elements appearing in the same containing element.
right
The right-hand side of the box will not touch elements appearing in the same containing element
both
Neither the left nor right-hand sides of the box will touch elements appearning in the same containing element.
none
Elements can touch either side

Screen Sizes, Resolution, and Page Size

Different visitors to your site will have different size screens that show different amounts of information, so your design neds to be able to work on a range of different sized screens. Screen size is important because it affects how big or small a user can make their windows and how much of the page they will see.

Resolution refers to the number of dats a screen shows per inch. Some devices have a higher resolution than desktop computers and most OS allow users to adjust the resolution of their screens. It is interesting to note that the higher resolution the smaller the text appears. Many mobile devices have screens that are higher resolutions than desktops.

Because screen sizes and display resolutions vary so much, web designers often try to create pages of around 960-1000 pixels wide. Judging the height that people are likely to see on the screen without scrolling down the page is much harger. Many designers used to assume that usrs would see the top 570-600 pixels of a page without scrolling, which used to be referred to as "above the fold". However as desktop screens have increased and mobile devices have become popular, the area users see is far more variable. Many usability studies have shown that a user is able to judge a website relevance in under a minute so it is important to communicate what the site is about at the top. But make sure to avoid clutter.

Fixed Width Layouts

Fixed width layout designs do not change size as the user increases or decreases the size of their browsers window. Measurements tend to be given in pixels.

Advantages

  • Pixel values are accurate at controlling the size and position of elements
  • The designer has far greater control over the apperance and position of items on the page than with liquid layouts
  • The size of an image will always remain the same relative to the rest of the page

Disadvantages

  • You can end up with big gaps around the edge of a page
  • If the user's screen is a much higher resolution than the designer's screen, the page can look smaller and text can be harder to read
  • If auser increases font sizes, text might not fit into the allotted spaces
  • The design works best on devices that have a site or resolution similar to that of desktop or laptop computers
  • The page will often take up more vertical space than a liquid layout with the same content

Liquid Layouts

Liquid layout designs stretch and contract as the users increases or decreases the size of their browser window. They tend to user percentages.

Advantages

  • Pages will expand to fill the enitre browser window so there are no spaces around the page on a large screen.
  • If the user has a small window, the page can contract to fit it without the user have to scroll to the side
  • The design is tolerant of users setting font sizes larger than the designer intended because the page can stretch

Disadvantages

  • If you do not control the width of sections of the page then the design can look very different than you inteded, with unexpected gaps around certain elements or items squashed together
  • If the user has a wide window, lines of text can become very long, which makes them harder to read
  • If the user has a very narrow window, words may be squashed and you can end up with few words on each line
  • If a fixed width item (like an image) is in a box that is too small to hold it (because the user has made the window smaller) the image can overflow over the text
